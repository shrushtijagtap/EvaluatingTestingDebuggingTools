15    /**
16    * This {@link Implementation} invokes a default method for the methods it instruments.
17    * A default method is potentially ambiguous if a method of identical signature is defined in several interfaces.
18    * For this reason, this implementation allows for the specification of <i>prioritized interfaces</i> whose default
19    * methods, if a method of equivalent signature is defined for a specific interface. All prioritized interfaces are
20    * searched for default methods in the order of their specification. If no prioritized interface defines a default method
21    * of equivalent signature to the given instrumented method, any non-prioritized interface is searched for a suitable
22    * default method. If exactly one of those interfaces defines a suitable default method, this method is invoked.
23    * If no method or more than one method is identified as a suitable default method, an exception is thrown.
24    * <p>&nbsp;</p>
25    * When it comes to default methods, the Java programming language specifies stronger requirements for the
26    * legitimacy of invoking a default method than the Java runtime. The Java compiler requires a method to be
27    * the most specific method in its defining type's type hierarchy, i.e. the method must not be overridden by another
28    * interface or class type. Additionally, the method must be invokable from an interface type which is directly
29    * implemented by the instrumented type. The Java runtime only requires the second condition to be fulfilled which
30    * is why this implementation only checks the later condition, as well.
31    */
