21    * An interface describing a <a
22    * href="http://martinfowler.com/bliki/CircuitBreaker.html">Circuit Breaker</a> component.
23    * </p>
24    * <p>
25    * A <em>circuit breaker</em> can be used to protect an application against unreliable
26    * services or unexpected load. It typically monitors a specific resource. As long as this
27    * resource works as expected, it stays in state <em>closed</em>, meaning that the
28    * resource can be used. If problems are encountered when using the resource, the circuit
29    * breaker can switch into state <em>open</em>; then access to this resource is
30    * prohibited. Depending on a concrete implementation, it is possible that the circuit
31    * breaker switches back to state <em>closed</em> when the resource becomes available
32    * again.
33    * </p>
34    * <p>
35    * This interface defines a generic protocol of a circuit breaker component. It should be
36    * sufficiently generic to be applied to multiple different use cases.
37    * </p>
38    *
39    * @param <T> the type of the value monitored by this circuit breaker
40    * @since 3.5
41    */
